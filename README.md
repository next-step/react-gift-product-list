- `react-gift-order` 저장소 fork 및 clone 진행하기
- 기존의 `react-gift-login` 작업 코드 옮겨 오기
- step1를 위한 브랜치 생성하기
- 요구사항 기록을 위한 `README.md` 작성


## 해야 할 일 (1단계)
- 본인만의 기준으로 **일관된 코드**를 작성하기
- **기능 단위**로 나누어 커밋하기

#### 로그인: ID 요구사항
- `ID`는 반드시 입력되어야 하고 이메일 형식을 충족하도록 설계하기
  - `ID input field`를 벗어났을 때 위 조건이 충족되지 않으면 적절한 에러 메세지를 출력하기
  - 조건이 충족되면 에러 메세지가 사라지도록 설계하기

#### 로그인: Password 요구사항
- `PW`는 반드시 입력 되어야 하고 최소 8글자 이상 입력되도록 설계하기
  - `PW input field`를 벗어났을 때, 이 조건이 충족되지 않으면 적절한 에러 메세지를 출력하기

#### 로그인: 버튼
- ID의 조건과 PW의 조건이 **모두 충족되었을 때**만 버튼이 활성화되도록 설계하기

#### 리팩터링
- 재사용을 고려해 Custom Hook을 사용하여 로그인 기능을 리팩터링하기


## 해야 할 일 (2단계): 주문하기 페이지 구현하기

## 목표
✔️ 쿠키 또는 세션 스토리지, Context API를 사용하여 로그인 정보를 관리
✔️ 주문하기 페이지 UI와, UI에 필요한 로직을 구현

#### 유념할 사항
- 재사용 가능성이 **높은** 컴포넌트, Hooks가 보이면 재사용성을 고려하여 만들기
- 별도의 From, Validation 라이브러리를 사용하지 않고 **useState로만 구현**하기

#### 로그인 정보
- 로그인 정보를 어디서든 사용할 수 있게 제작하기
- 새로고침을 하더라도 로그인 정보가 유지되도록 하기
- 로그인 한 유저는 로그인 버튼 클릭 시 마이페이지(`/my`)로 연결되도록 하기
- 마이페이지는 로그인한 사람만 접근 가능하도록 설계하기
- 로그인 하지 않은 사람이 접속하면 로그인 페이지로 연결하기
- 마이페이지에서 로그아웃을 할 수 있고, 로그아웃 시 로그인(`/login`) 페이지로 연결되도록 하기

#### 주문하기
- 상품 아이템을 클릭하면 주문하기 페이지(`/order`)로 연결되도록 하기
- 만약 로그인이 되어있지 않다면 로그인을 유도하기
- 로그인 성공 시 주문하기 페이지(`/order`)로 접근 가능하게 하기
- 버튼 클릭 시 아래 유효성 검사 조건이 충족되지 않으면 각 폼 필드에 안내 메시지를 보여주기
  - 메시지는 반드시 입력
  - `보내는 사람`과 `받는 사람` 이름을 반드시 입력
  - 받는사람 전화번호를 반드시 입력
  - ‘`01012341234`’와 같이 전화번호 규칙에 맞게 입력
  - 수량은 1개 이상

## 해야 할 일 (3단계): 단체 주문 기능 구현하기

#### 주문하기 페이지
- 주문하기 페이지의 폼을 React Hook Form를 사용해서 리팩터링하기 (form 데이터를 useState로 다루는 경우는 없도록 하기)

#### 받는 사람 기능 고도화
- 받는 사람을 최대 10명까지 등록 가능
- 10명의 정보가 정확하게 입력되지 않으면 정보가 반영되지 않음
- 받는 사람의 전화번호가 중복되면 안됨
- 최소 선택 수량은 1개 이상이어야 함
- 전화번호는 01012341234 형태로만 받을 수 있음

#### 주문하기 페이지 고도화
- 예시(참고) 사이트를 살펴보고 변경된 기능을 구현하기
- (선택) 유효성 검사를 Zod 라이브러리를 사용해서 처리하기
