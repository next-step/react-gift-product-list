## 2단계 과제 구현 내용

- 로그인 시 /login api, POST 메서드 이용하여 로그인을 할 수 있도록 구현했습니다.
- 로그인 시 응답으로 받는 정보를 전역상태로 관리하여 토큰, name등을 다른 컴포넌트에서 사용할 수 있도록 구현했습니다.
- 로그인 시 api 에러가 나면 toast 라이브러리를 활용한 에러메시지를 출력하도록 했습니다.
- 주문페이지가 렌더링될 때 제품정보를 /products/:productId/summary 를 이용하여 가져와서 출력하도록 했습니다.
- 제품정보를 가져올 때 api에러가 나면 toast에러메시지를 출력하고 홈화면으로 돌아가도록 했습니다.
- 주문페이지에서 보내는 사람 인풋필드를 username으로 defaultvalue 설정하여 기본으로 채워져있도록 했습니다.
- 주문페이지에서 주문 폼 제출시 /order api요청으로 주문을 제출하도록 했습니다.
- 주문 제출 시에 401에러가 나면 관련 toast 에러메시지 출력 후 로그인페이지로 이동하게 만들었습니다.
