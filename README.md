# react-gift-order

## step0

- 미션 1 피드백 반영
  - 라우트 확인에 useLocation 사용하도록 수정
  - 아이콘에서 네비게이션 로직 분리
  - 이전 히스토리가 없을 때 navigate(-1) 동작 막기

## step1

- 아이디(반드시 입력, 이메일 형식)
  - 만약 ID Input Field를 벗어났을 때 위 조건이 충족되지 않으면 에러 메시지 출력
  - 위 조건이 충족되면 에러 메시지가 사라짐
- 비밀번호 (8글자 이상)
  - 만약 PW Input Field를 벗어났을 때 위 조건이 충족되지 않으면 에러 메시지 출력
- 로그인 버튼은 ID, PW 조건이 모두 만족일 때만 활성화

## step2

- 로그인 정보
  - 로그인 정보를 어떤 페이지에서든 사용 가능하도록 함
  - 새로고침 하더라도 로그인 정보 유지
  - 로그인 하고 나면 로그인 버튼 클릭 시 마이 페이지로 이동
    - 로그인 한 사람만 접근 가능, 로그인 하지 않았다면 로그인 페이지로
- 주문하기
  - 상품 아이템을 클릭하면 주문하기 페이지로 이동
  - 로그인을 하지 않았으면 로그인을 유도하고 로그인 시 주문하기 페이지로 접근
  - 주문하기 기능 구현(각 폼 필드에 안내메시지 출력)

## step3

- step2 피드백 반영
  - 없는 상품 주문하기 페이지로 이동시 not found 페이지로 이동
  - sessionStorage 커스텀 훅으로 분리하여 사용
- 주문하기 페이지의 form을 React Hook Form을 사용해서 리팩토링
- 주문하기 페이지, 받는 사람 기능 고도화
  - 받는 사람 버튼 클릭시 받는 사람을 추가하는 모달 표시
  - 모달 내에서 추가하기 클릭시 받는 사람 form 생성(10명 까지)
  - 완료 클릭시 받는 사람 목록을 표시하고 수정 버튼을 통해 수정 가능
